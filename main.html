Типи Даних у JavaScript поділяються на дві категорії: примітивні та об'єктні.

Примітивні типи

  • string - (const string = "Hello World";)
  • number - (const num = 777;)
  • boolean - (const weArePotatoes = true;)
  • undefined - це наприклад не оголошена змінна(let x;)
  • null - вказує на відсутність значення (let x;)
  • bigint - (const bigNumber = 12345678901234567890123456789012345678910n;)
  • symbol - (Symbol('description');)

Об'єктні Типи Даних

Включають в себе об'єкти (object), масиви (array), функції (function) та інші. Вони володіють складною структурою та передаються за посиланням, що означає, що при копіюванні створюється посилання на той самий об'єкт.

Danger zone ⚠

typeof null; // "object" null – це "нічого", але повертається "object" через історичну помилку в JavaScript, яку вже не виправляють для сумісності.
1 + '2'; // "12" Додавання числа і рядка
[] + []; // ""
[1, 2] + [3]; // "1,23"		 масиви автоматично перетворюються у рядки
false == 0; // true
false === 0; // false		 == виконує неявне приведення типів.
=== — строге порівняння без приведення
NaN === NaN; // false NaN (Not-a-Number) — єдиний тип у JS, який не дорівнює сам собі.
[] == false; // true Порожній масив → '' (рядок) → 0 → false, от і [] == false → true.
undefined + 1; // NaN undefined не перетворюється на число напряму → результат NaN
true + true; true → 1, тому 1 + 1 = 2
{} + []; // 0
[] + {}; // "[object Object]"		 Синтаксис {} на початку рядка трактується як блок коду, а не об'єкт.

⚠Null має тип "object" у JavaScript з історичних причин. Коли JavaScript був розроблений на початку 1990-х, помилка була допущена в дизайні мови, і тип даних "null" був помилково визначений як "object". Ця помилка залишилася в мові для забезпечення сумісності з існуючими програмами. Тобто тип "object" для null - це особливість мови JavaScript, яку потрібно пам'ятати при роботі з цим значенням.
